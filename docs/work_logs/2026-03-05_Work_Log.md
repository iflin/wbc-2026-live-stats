# 工作日誌：2026 WBC 即時比賽資料抓取與處理 (台灣 vs 澳洲)

**日期**：2026-03-05
**專案**：2026 WBC 賽事數據追蹤
**目標**：建立即時抓取工具，即時取得比賽逐球紀錄，並經過 Statcast 數據計算，產出可供賽後報導、視覺化與深入分析使用的數據快照。

---

## 📅 今日工作總結 (Executive Summary)

今日成功建立了基於 R 語言與 `baseballr` 的即時數據看板系統，針對正在進行中的「台灣 vs 澳洲」賽事進行了深度資料探勘。此系統不仅能即時顯示數據，還能將其持續累積為易於閱讀的 Markdown 文件和利於設計的 CSV 結構檔。

### ✨ 核心實作功能與新增指標

開發團隊今日完成以下核心進階擊球與投球指標的即時計算：

1. **打擊火力分析**：
   - 每隊最快擊球初速 (Max Exit Velocity)
   - 每一位打者的平均擊球速度 (Avg EV) 與仰角 (Avg LA)
   - 針對 **「hit in play」** 的進階數據：包含該球的對應仰角、次數與平均初速。
   - 所有上壘與全壘打事件的初速與仰角紀錄。
2. **賽後統計報表生成**：
   - 各隊團隊的三振與四壞球次數 (Team K & BB) 與團隊打擊結果分佈。
   - 各打者的詳細出局與安打結果統計。
   - **投手深度分析**：
     - 各投手的球種使用比例、平均球速、平均轉速與總用球數。
     - **投手最快球速**、轉速與使用球種擷取。
     - **好壞球比例 (Strike/Ball Ratio)**：精確掌控投手控球穩定度。
     - **揮空率 (Whiff Rate) 模型建立**：包含全場變速球揮空率、打者對變速球揮空率、各投手總揮空率、各投手依「球種」細分的揮空率。
3. **主審判決熱區圖 (Umpire Heatmap)**：
   - 導入以客觀座標 (`pX`, `pZ`) 為基礎的主審好壞球判決熱區圖，使用六邊形分箱 (Hexbin) 呈現。
   - 成功分離主、客隊打擊時的情境，可清晰比較主審在面對不同隊伍時，邊角好球帶判決是否存在差異。

## 💡 關鍵發現與除錯紀錄 (Insights & Debugging)

1. **即時覆寫與附加的平衡**：
   初期遇到 Markdown 報告內容重疊問題，後改為「**重啟覆寫最新結果 (Append = FALSE)**」的邏輯，而對於需供後方深入運算的 CSV 檔，則加上「**時間戳記**」進行斷點保存快照。
2. **"Ball In Dirt" 座標遺失的正確解讀**：
   在處理主審熱區圖時，發現總球數與進壘座標數量有 8 球的落差。經追蹤發現，這 8 顆是 `Ball In Dirt`（挖地瓜），Statcast 原本即不會/無法提供這類球精確的 `pX`/`pZ` 座標。我們將其保留在文字比例總計中，但在圖表可視化上將其合理排除。這確保了視覺輸出的準確性。

## 🚀 跨部門檢討與後續三日工作建議

接下來三天台灣隊仍有賽程，建議各角色根據今日建立的基礎，按以下方向優化工作流程：

### 🧑‍💻 給開發人員 (Engineering / Data Pipeline)

- **自動化穩定性**：目前的 `live_dashboard.sh` 使用無限迴圈定時抓取。建議未來三天可評估引入更健壯的 Cron Job 或是背景排程，同時增加 Log 機制，避免腳本因為單次 API 逾時而中斷。
- **快照清理機制**：目前每分鐘會產生一份帶時間戳記的 CSV 快照。當賽事較長時會佔用不少空間與數量，建議增加一個「賽後只保留局末或最終局資料」的清理指令。

### 📊 給數據分析者 (Data Analytics)

- **情蒐洞察**：今天產出的「各投手各球種揮空率」與「主審對兩隊不同好球帶熱區圖」是非常寶貴的素材。明日比賽可提前對比對手昨日的弱點球種，或者預判某位主審針對外角的嚴格程度。
- **打者發力點**：關注新增的 `Hit In Play` 數據，找出「打得出初速，但仰角老是負數（滾地球）」的打者，分析是否揮棒軌跡需要調整。

### 🎨 給 UI/UX 設計師與前端 (Design / UI)

- **色彩語意統整**：今日熱區圖已採用了色盲友善且語意明確的漸層（綠底紅字改為 #0D6CF2 低好球率 -> #FFFFFF -> #28BD73 高好球率）。後續製作互動式 Web Dashboard 時，建議這套色碼設定為 Design Token 統一使用。
- **手機端排版**：昨日已處理過 RWD 500px 的切換。請注意今日新增了大量深度的 Table (例如：各球種揮空率)，若要在前端網頁呈現，請考慮使用**折疊面板 (Accordion)** 或**水平滑動表格**，避免手機畫面過於雍長。

### ✍️ 給記者與 PM (Editorial / Management)

- **賽後戰報素材**：現在系統會在 `reports/` 下存放完整的統計報告。您可直接引述其中的「**好壞球比例**」與「**最快擊球初速紀錄**」作為賽後戰報的亮點。
- **主審話題性**：主審針對兩支隊伍的好球帶變化是一大看點。熱區圖直接支持了「判決尺度」的客觀論述，這將是賽後社群報導極佳的話題切入點。
